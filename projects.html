01. Automated CI/CD for Legacy Banking Middleware

GitHub Actions Docker Terraform AWS EKS

The Challenge

A legacy Java application required manual deployment to on-premise VMs, leading to a 2-day lead time for production releases and frequent "human-error" outages.

Design Decisions

Decision: Containerization over VM Image Baking

I chose to containerize the application using multi-stage Docker builds instead of using traditional Packer/AMI flows. Why? To standardize the environment between the developer's local machine and the production EKS cluster, reducing "it works on my machine" issues by 90%.

Decision: GitHub Actions for Orchestration

Despite the team's familiarity with Jenkins, I implemented GitHub Actions. Why? To eliminate the overhead of managing a Jenkins master server and to leverage "Jobs-as-Code" tightly coupled with the source repository.

Architecture Flow

The pipeline includes: Linting -> Unit Testing -> Security Scanning (Trivy) -> Build -> Push to ECR -> ArgoCD Sync.

The Outcome

  • Lead Time: Reduced from 48 hours to 12 minutes.
  • Reliability: 100% automated rollbacks implemented on failed health checks.
  • Cost: Switched to Spot Instances for Dev environments, saving 60% on monthly cloud spend.
